# string 类型（一）：类型基本介绍 \&\& 相关命令

# 一\. String 类型基本介绍

redis 中的所有 key 都是字符串，value 的类型是存在差异的\~

Redis 中的字符串，直接就是按照二进制数据的方式来存储的（**不会做任何的编码转换**，存的是啥，取出来还是啥）

不仅仅可以存储文本数据，还可以存整数，普通的文本字符串，Json，xml，二进制数据\(图片，音频，视频\)

但是音频视频的话，体积可能会比较大

Redis 对于 string 类型，限制了大小最大是 512 M

Redis 是单线程模型，希望进行的操作都能比较快速

所以我们不建议用 string 存音频视频



我们之前讲 MySQL 的时候，知道 MySQL 默认的字符集，是拉丁文，插入中文就会失败\~，Redis 就不管这么多了

一般来说，redis 遇到乱码问题的概率更小，但是两个地方用的编码方式不同可能也会出现这种问题，但是概率更小了
