**我们还有个比较重要的问题，之前说redis快！**

我们说到的 redis 的快，是相对于 MYSQL 这样的关系型数据库的\~

但是如果是直接和内存中的操作变量相比，就没有优势了，甚至更慢了



假设我们现在是一个单机系统，里面有一个应用服务器

比如，应用程序要存储一些数据

比如，存储一下用户点赞数

视频id，点赞个数，键值对格式来存储

那么是用一个redis来存储，还是直接在内存中搞个 hash map 来存呢？

![Image](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=NzgzZTRhYWZmZDNlMDNmOTFiNjFhZjY4MTAzNmQwMGJfMDhmMTJmZTg1NWI4MDQ4NDgwMDQyNTFhM2U3NDkzYWFfSUQ6NzY1NjEzMTQxOTEzOTI5NjQ4MF8xNzgyNTgyMzMzOjE3ODI2Njg3MzNfVjM)

使用 hash map 是直接操作内存

使用 redis 是先通过网络，再操作内存（即使是很好的网卡也是不如直接操作内存的）

上述场景中，是否需要使用redis，要结合实际的需求来确定

引入redis的缺点，会更慢，但是有了redis之后，就可以把谁单独存储，后续应用服务器重启，不会影响数据内容
